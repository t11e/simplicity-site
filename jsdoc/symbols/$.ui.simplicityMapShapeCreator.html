Namespace $.ui.simplicityMapShapeCreator

A map shape creator for use in Discovery searches.

This widget is generally only called through one of its implementations that are specific to a map provider. Current simplcityMapShapeCreator implementations:

  • simplicityGoogleShapeCreator
  • simplicityNokiaMapShapeCreator
  • simplicityMapQuestShapeCreator

A widget which creates a single shape on a map. Supported shapes are Point, MultiPoint, LineString and Polygon. Whenever the shapes change, the underlying state input element will be updated.

The widget supports radius values by showing circles around any drawn point, as long as the point is not part of polygon. The radius value can be used for exactDistance calculations in a Discovery geoloc type query.

Requires an instance of simplicity<mapprovider>ShapeCreator implementation. When used in conjunction with simplicityMapShapeCreatorUi, a complete visible shape creator interface is made available.

Triggers a simplicitymapshapecreatorshapechange event which other components can use to add additional functionality.

The map is formatted into geoJson (http://www.geojson.org/geojson-spec.html) as follows: 
{
  "placemarks": {
    "type": "Polygon",
    "coordinates": [[[lng1,lat1], [lng2, lat2], [lng3, lat3], [lng1, lat1]]]
  },
  "properties": {
    "radius": 1.5
  }
}

The Discovery Search Engine criterion using the geoJson format would look like this:

{
   "dimension": "location",
   "placemarks": {
     "type": "Polygon",
     "coordinates": [[[lng1,lat1], [lng2, lat2], [lng3, lat3], [lng1, lat1]]]
   },
   "exactDistance": 1.5
}

<private> <static> {array} $.ui.simplicityMapShapeCreator._getLineStringBufferPoints()
Buffers a line string the radius in play. The LineString will be buffered into a Polygon in which the end points and connecting vertices will be capped and shaped to represent the true shape of the buffered LineString. The shape coordinates are built in two arrays, the "outbound" (out) and the "inbound" (back). Each "outbound" buffered points are calculated from the heading of the vertex to the next vertex traveling radius distance at 90 degrees from that heading. The "outbound" shape includes the encap arc at the last vertex. Each "inbound" buffered points are calculated from the heading of the vertex to the previous vertex traveling radius distance at -90 degrees from that heading. The "inbound" shape includes the encap arc at the first vertex. For points between the first and the last (connector), the curve shape is either a point or an arc. The curve shape is an arc for obtuse angles. For acute angles, it is the intersection of two lines, one traveling with heading from the previous vertex to the connector, the the other traveling from the next vertex back to the connector. Both of these lines originate from their respective buffered points. The information used to create the buffer is stored in this._vertexData;
Returns:
{array} An array of coordinates for the Polygon representing the buffered LineString.
